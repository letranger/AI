1. What is Data Mining

Discover knowledge from data, or, knowledge discovery from data (KDD).

Extraction of interesting, non-trivial, implicit, previously unknown and potentially useful patterns of knowledge from huge data.

簡單來說，資料探勘就是從大量資料中，找出隱藏的、有價值的規律或知識。這些規律通常不是靠人工就能輕易發現的，需要透過演算法自動分析。例如：超市從數百萬筆消費記錄中發現「買尿布的人常常也會買啤酒」，這就是資料探勘的經典案例。

2. Why Data Mining

雖然資料量大，取得容易，但是其中夾雜太多無關資料，光靠人力根本無法處理。

想像一下：

  • 全球每天產生超過 25 億 GB 的資料，包含社群貼文、購物記錄、感測器數據等
  • 一間超市每天可能產生數萬筆交易記錄，一年就是上千萬筆
  • 醫院累積了數十年的病歷資料，其中可能隱藏著疾病之間的關聯

這些資料量遠遠超過人工逐一檢視的能力，因此我們需要自動化的方法——也就是資料探勘技術——來從中找出有意義的模式與知識。

6. General Types of Data Mining

6.1. Association rule mining（關聯規則）

找出資料之間的伴隨關係，也就是「買了 A 的人通常也會買 B」這種規律。

  • a → b
  • 由大賣場的銷售記錄，發現買了 a 的人都買了 b。決策: 把兩種商品擺在一起
  • 某種資訊出現 → 下雨機率高
  • 應用範例：超市商品擺設、網站推薦系統（「買了這個的人也買了…」）

6.2. Classification（分類）

根據已知的標籤資料，學習出一套規則，用來預測新資料的類別。就像老師批改了一堆考卷（已知答案），學生從中學到判斷規則。

  • 範例：已知六個人是否購買某商品（Y=買, N=不買）
    • A:Y, B:N, C:Y, D:Y, E:N, F:Y
  • 電腦會根據這些資料建立一棵 decision tree（決策樹），用來預測第七個人會不會購買
  • 應用範例：垃圾郵件偵測（判斷一封信是不是垃圾信）、疾病診斷

6.3. Clustering（分群）

把相似的資料自動歸為一組，不需要事先知道有哪些類別。電腦會自己找出資料之間的相似性。

  • 範例：將六個人依相似度自動分成三組
    • {A, B}：彼此相似，歸為一組
    • {C, D, E}：彼此相似，歸為一組
    • {F}：和其他人都不太一樣，自成一組
  • 應用範例：顧客分群（VIP 客戶 vs 一般客戶）、新聞文章自動分類

7. Association rule mining

在關聯規則分析中，有兩個最重要的指標：

Support（支持度） ：兩個商品「同時被購買」的次數。

Confidence（信賴度） ：一個條件機率。白話說就是：「買了 A 的人裡面，有多少比例也買了 B？」

公式： 

confidence(A → B) = A 和 B 同時出現的次數 ÷ A 出現的次數

以下面的交易資料為例，假設我們要算 B → C 的 confidence：

  1. 先數 B 出現在哪些交易：100, 200, 400 → 共 3 次
  2. 再數 B 和 C 同時出現在哪些交易：200, 400 → 共 2 次
  3. confidence(B → C) = 2 ÷ 3 ≈ 0.667

意思是：買了 B 的人中，有 66.7% 也買了 C。

反過來算 C → B 呢？

  1. C 出現在哪些交易：200, 400 → 共 2 次
  2. C 和 B 同時出現：200, 400 → 共 2 次
  3. confidence(C → B) = 2 ÷ 2 = 1.0

意思是：買了 C 的人，*100%* 都買了 B。注意 B → C 和 C → B 的 confidence 不一樣！

我們會設定門檻值來過濾規則，只保留夠強的：

  • min_support = 2（至少同時出現 2 次）
  • min_conf = 2/3（信賴度至少 66.7%）
  • 同時達到兩個門檻的規則，叫做 Strong rule

7.2. Frequent item sets（頻繁項目集）

找出頻繁項目集的步驟如下：

  1. 先列出所有可能的商品組合（注意：這裡是「組合」而非「排列」，{A,B} 和 {B,A} 視為同一組）
  2. 計算每種組合在交易資料中同時出現的次數（即 support）
  3. 只保留 support >= min_support 的組合

以上面的交易資料為例：

  • 1-itemset（單一商品）：{A}, {B}, {C}, {D}… 先計算各自出現次數
  • 2-itemset（兩兩組合）：{B,C}, {A,B}, {A,D}, {B,D}… 計算同時出現的次數，只保留 >= min_support 的
  • 3-itemset：{A,B,D}… 依此類推

接下來要從頻繁項目集產生關聯規則：

  • 對於 {B,C}，有兩個可能的規則：B → C 或 C → B，兩個都要分別計算 confidence
  • 對於 {A,B,D}，可能的規則更多：A → {B,D}、B → {A,D}、{A,B} → D… 每一種都要檢查

7.3. Strong rules (同時符合 min_support, min_conf)

同時滿足 support >= 2 且 confidence >= 2/3 的規則才是 Strong rule：

  • B → C：support=2, confidence=2/3≈0.667（B 出現 3 次，其中 2 次有 C）✓
  • C → B：support=2, confidence=2/2=1.0（C 出現 2 次，2 次都有 B）✓
  • {A,D} → B：support=2, confidence=2/3≈0.667（{A,D} 一起出現 3 次，其中 2 次有 B）✓

目標就是讓電腦自動把所有 Strong rules 找出來。

7.4. 產生 Strong rules 用到的演算法: Apriori Algorithm

暴力法是把所有商品的所有組合都列出來，一一計算 support——但商品一多，組合數量會爆炸（10 種商品就有 1023 種可能的組合）。

Apriori 演算法的核心想法：*由小到大，逐層篩選*，淘汰掉的就不再往下展開。

步驟如下：

  1. 先數所有 1-itemset 的 support，淘汰不達標的
  2. 用「活下來」的 1-itemset 組合出候選 2-itemset，再數 support，再淘汰
  3. 用「活下來」的 2-itemset 組合出候選 3-itemset⋯⋯
  4. 重複直到沒有新的頻繁項目集為止
  5. 最後從所有頻繁項目集產生規則，只保留 confidence 達標的

為什麼這樣有效？因為 Downward Closure Property：如果 {A, B} 不夠頻繁，那 {A, B, C} 一定也不頻繁——直接跳過，不用算。這一招讓 Apriori 在實務上比暴力法快非常多。

從結果可以觀察到：

  • B → C 的 confidence 為 2/3，代表買了 B 的人有 2/3 的機率也會買 C
  • C → B 的 confidence 為 2/2 = 1.0，代表買了 C 的人 一定 也買了 B
  • 兩條規則的 support 相同（都是 2），但 confidence 不同，這說明關聯規則是有方向性的

8. Downward Closure Property（向下封閉性質）

  • Any subset of a frequent itemset must be frequent
  • If there is any itemset which is infrequent, its superset should not be frequent

白話說：如果某個商品組合不夠常見（不是頻繁項目集），那麼包含它的更大組合一定也不常見，就不需要再檢查了。

舉例：如果 {啤酒, 尿布} 只出現 1 次（不夠頻繁），那 {啤酒, 尿布, 牛奶} 最多也只會出現 1 次，一定也不頻繁。這個性質大幅減少了需要計算的組合數量，是 Apriori 演算法能有效運作的關鍵。

9. Sequential Pattern Mining（序列模式探勘）

9.1. Mining sequential implications between items

和關聯規則不同，序列模式探勘強調事件發生的「先後順序」。

表格記號說明：

  • < ... > 表示一個完整的序列（按時間先後排列）
  • (1) 表示在某個時間點購買了商品 1
  • (2,5) 表示在同一個時間點同時購買了商品 2 和 5
  • 整個序列由左到右代表時間的推進
SID Sequences
100 < (1) (2,5) (3) (4) >
200 < (1) (3) (4) (2) (3,5) >
300 < (1,3,5) (4) (2) (3) >
400 < (1) (3) (5) >
500 < (4) (3,5,6) >

以 SID=100 為例：顧客先買了商品 1，接著同時買了商品 2 和 5，之後買了商品 3，最後買了商品 4。

生活化例子： 顧客先買了手機 → 過幾天買了手機殼 → 再過一段時間買了螢幕保護貼。這個「手機 → 手機殼 → 保護貼」就是一個序列模式。

  • min_support = 2
  • Some frequent sequential patterns:

{1,4,3,2}, {1,3,5},{4,5}

  • 這些模式代表有至少 2 位顧客的購買順序符合這個序列

11.1.1. 計算提醒

假設牛奶出現 5 次，牛奶和麵包同時出現 4 次：

  • 牛奶 → 麵包 的 confidence = 4 ÷ 5 = 0.8（買牛奶的人有 80% 也買麵包）
  • 反過來，如果麵包出現 6 次，則 麵包 → 牛奶 的 confidence = 4 ÷ 6 ≈ 0.667
  • 同一組商品，方向不同，confidence 就不同！

  1. 將交易資料轉換為交易矩陣（DataFrame），每一列是一筆交易，每一欄是一種商品，1 代表有買、0 代表沒買：

  1. 計算每個商品的出現次數（support count）
  2. 找出所有 support >= 3 的 2-itemset
    • 2-itemset = 兩種商品的組合（例如 {牛奶, 麵包}）
    • support >= 3 = 這兩樣東西在 8 筆交易中「同時出現至少 3 次」
    • 商業意義：如果兩個商品很少被一起買（例如只有 1 次），那就算分析出規則也沒用——可能只是巧合。設定 support 門檻是為了濾掉這種雜訊，只留下真的夠常見、值得拿來做決策的組合。
  3. 對這些 2-itemset 計算雙向的 confidence，找出 confidence >= 0.5 的 strong rules
    • 算法：confidence(A → B) = A 和 B 同時出現的次數 ÷ A 出現的次數
    • 例如：牛奶出現 5 次，{牛奶, 麵包} 同時出現 4 次 → confidence(牛奶→麵包) = 4/5 = 0.8
    • 意義：「買了牛奶的人裡面，有多少比例也買了麵包？」答案是 80%
    • 為什麼要算「雙向」？因為 牛奶→麵包 和 麵包→牛奶 的 confidence 不同。前者問的是「牛奶帶動麵包的能力」，後者問的是「麵包帶動牛奶的能力」——方向不同，結論就不同
    • confidence >= 0.5 表示：至少有一半的人買了 A 也會買 B，這個關聯才夠強，值得拿來做促銷決策

11.3.1. 觀念提醒：Support 與 Confidence

在分析之前，先搞懂兩個關鍵數字（搞不懂的話，物流中心見）：

  • Support（支持度） ：兩個商品在所有交易中「同時被購買」的次數。例如 71 筆交易中有 13 筆同時買了鮮奶和麵包，則 support = 13。support 越高 = 這組合越常出現。
  • Confidence（信賴度） ：已知顧客買了 A，他「也買了 B」的比例。例如買泡麵的 12 人中有 11 人也買了茶葉蛋，則 泡麵 → 茶葉蛋 的 confidence = 11/12 ≈ 92%。confidence 越高 = A 對 B 的帶貨力越強。
    • 注意：=A → B= 和 B → A 的 confidence 通常不同！買泡麵的人幾乎都買茶葉蛋，但買茶葉蛋的人不一定買泡麵（也許他只是肚子餓）。

11.3.2. 情境說明
